## ١. التثبيت

يتطلب apython بايثون 3.11 أو أحدث. من مجلد المستودع:

```bash
git clone https://github.com/GalaxyRuler/apython
cd apython
pip install -e .
```

هذا يثبّت الأمر `apython` في بيئتك. تحقق من التثبيت:

```bash
apython --version
```

لديك الآن أربع طرق لتشغيل شيفرة apython، تعكس ما يفعله `python` تماماً:

```bash
apython script.apy [وسائط...]   # تشغيل ملف
apython -c 'اطبع("مرحبا")'       # تشغيل سطر مباشرةً
apython - < script.apy           # القراءة من المدخل القياسي
apython                           # الوضع التفاعلي (REPL)
```

رموز الخروج: `0` نجاح، `1` خطأ في التشغيل أو الترجمة، `2` خطأ في الاستخدام.

## ٢. مرحباً بالعالم

احفظ هذا في ملف باسم `مرحبا.apy`:

```arabic
اطبع("مرحباً، يا عالم")
```

شغّله:

```bash
$ apython مرحبا.apy
مرحباً، يا عالم
```

`اطبع` هي الاسم العربي لدالة `print` في بايثون. كل دالة مدمجة وكل كلمة مفتاحية في البلاغ العربي تعمل بالطريقة ذاتها تماماً — apython هو بايثون بمفرداتٍ عربية، لا أكثر.

القائمة الكاملة لكل الرموز المترجمة موجودة في
[`dictionaries/ar-v1.md`](../../dictionaries/ar-v1.md).
لا حاجة لحفظها كلها؛ الأمثلة التالية تُعرّفك بالكلمات المفتاحية تدريجياً.

## ٣. المتغيرات والحساب

إسناد المتغيرات يعمل كما في بايثون تماماً — apython يترجم فقط الأسماء *المدمجة*، وليس أسماءك الخاصة. اختر أي معرّف عربي تريده:

```arabic
العمر = 25
السنوات_حتى_التقاعد = 65 - العمر
اطبع(f"باقي {السنوات_حتى_التقاعد} سنة حتى التقاعد")
```

المخرجات:

```
باقي 40 سنة حتى التقاعد
```

**ملاحظات مهمة:**

- **المتغيرات متعددة الكلمات تستخدم `_`** وليس الفراغ، لأن بايثون يعامل الفراغ كحد فاصل بين الرموز. هذه القاعدة ثابتة في كل apython.

- **السلاسل الحرفية المنسقة (f-strings) تعمل بشكل طبيعي.** المعرّفات داخل الأقواس المعقوصة تُترجَم تماماً كبقية الشيفرة.

- **أرقام عربية-هندية مدعومة تلقائياً.** كتابة `العمر = ٢٥` تساوي تماماً `العمر = 25`. خلط منظومتَي ترقيم في رقم واحد (مثل `١2`) يُنتج خطأ نحوياً واضحاً.

- **علامات الترقيم العربية تُحوَّل تلقائياً** خارج النصوص: `،` تصبح `,`، و`؛` تصبح `;`، و`؟` تصبح `?`. أما داخل النصوص فكل حرف عربي يبقى كما هو.

انظر: [`examples/02_arithmetic.apy`](../../examples/02_arithmetic.apy)

## ٤. التحكم في التدفق

الكلمات المفتاحية للتحكم في التدفق:

| بايثون | apython | المعنى |
|---|---|---|
| `if` | `إذا` | "إذا" |
| `else` | `وإلا` | "وإلا / وفي غير ذلك" |
| `elif` | `وإلا_إذا` | "وإلا إذا" |
| `for` | `لكل` | "لكل / من أجل كل" |
| `while` | `طالما` | "طالما / ما دام" |
| `in` | `في` | "في" |
| `break` | `اكسر` | "اكسر الحلقة" |
| `continue` | `استمر` | "استمر / واصل" |
| `range` | `نطاق` | "نطاق / مدى" |

حلقة مع شرط في داخلها:

```arabic
لكل عدد في نطاق(1, 6):
    إذا عدد % 2 == 0:
        اطبع(f"{عدد}: زوجي")
    وإلا:
        اطبع(f"{عدد}: فردي")
```

المخرجات:

```
1: فردي
2: زوجي
3: فردي
4: زوجي
5: فردي
```

المسافة البادئة مهمة في apython تماماً كما في بايثون — أربعة مسافات لكل مستوى بالاتفاق.

انظر: [`examples/03_control_flow.apy`](../../examples/03_control_flow.apy)

## ٥. الدوال

`def` تصبح `دالة` (الكلمة الرياضية الفصيحة لـ "function"). و`return` تصبح `ارجع`. القيم الافتراضية للمعاملات تعمل بالطريقة المعتادة:

```arabic
دالة جمع(أ, ب=10):
    ارجع أ + ب

اطبع(جمع(5))      # 15
اطبع(جمع(5, 7))   # 12
```

أسماء الدوال والمعاملات والمتغيرات المحلية من اختيارك — apython لا يترجمها، فيمكنك استخدام أي معرّف عربي أو إنجليزي أو مختلط صالح في بايثون.

**ملاحظة عن تطبيع المعرّفات:** يطوي apython بعض المتغايرات الإملائية العربية قبل البحث عنها، فـ`قيمه` و`قيمة` (بالتاء المربوطة) يُحلّان إلى الاسم ذاته. القواعد الكاملة في
[`decisions/0004-normalization-policy.md`](../../decisions/0004-normalization-policy.md)؛
في الشيفرة العادية يمكنك تجاهل التفاصيل.

انظر: [`examples/04_functions.apy`](../../examples/04_functions.apy)

## ٦. هياكل البيانات

القوائم والقواميس والصفوف والمجموعات كلها تعمل بشكلها المعتاد. أسماء الأنواع مترجمة (`قائمة`، `قاموس`، `صف`، `مجموعة`) لكنك نادراً ما تستدعيها مباشرةً — معظم الشيفرة تستخدم الصياغة الحرفية:

```arabic
الفواكه = ["تفاح", "موز", "برتقال"]
الأسعار = {"تفاح": 3, "موز": 2, "برتقال": 4}

لكل فاكهة في الفواكه:
    اطبع(f"{فاكهة}: {الأسعار[فاكهة]} ريال")
```

المخرجات:

```
تفاح: 3 ريال
موز: 2 ريال
برتقال: 4 ريال
```

التوابع الشائعة لها أسماء عربية:

| التابع (بايثون) | الاسم العربي |
|---|---|
| `.append` | `اضف` |
| `.pop` | `انتزع` |
| `.keys` | `مفاتيح` |
| `.values` | `قيم_القاموس` |
| `.items` | `عناصر` |
| `.get` | `اجلب` |
| `.add` | `ضم` |
| `.union` | `اتحاد` |
| `.split` | `قسم` |
| `.join` | `اجمع` |
| `.replace` | `استبدل` |
| `.upper` | `كبير` |
| `.lower` | `صغير` |

القائمة الكاملة للتوابع في القسم السادس من
[`dictionaries/ar-v1.md`](../../dictionaries/ar-v1.md).

## ٧. الأصناف

`class` تصبح `صنف`. المعامل التقليدي `self` يُسمّى `الذات` بالاتفاق. التوابع الخاصة مثل `__init__` تبقى بأسمائها الإنجليزية — فهي أسماء بروتوكول يحددها بايثون وليست مفردات يراها المستخدم.

```arabic
صنف نقطة:
    دالة __init__(الذات, س, ص):
        الذات.س = س
        الذات.ص = ص

    دالة المسافة_من_الأصل(الذات):
        ارجع (الذات.س ** 2 + الذات.ص ** 2) ** 0.5

نقطة_1 = نقطة(3, 4)
اطبع(نقطة_1.المسافة_من_الأصل())   # 5.0
```

الوراثة و`super()‎` (أو `الأصل()‎` بالعربية) والتوابع الصنفية والتوابع الثابتة كلها تعمل بالطريقة ذاتها في بايثون — ترجم الكلمة المفتاحية واكتب الباقي بالأسلوب المعتاد.

انظر: [`examples/06_classes.apy`](../../examples/06_classes.apy)

## ٨. الاستيراد

يُثبّت apython خطّافاً في `sys.meta_path` يجعل ملفات `.apy` قادرةً على استيراد بعضها بشفافية تامة:

```arabic
# مساعد.apy
دالة مربع(عدد):
    ارجع عدد ** 2

دالة مكعب(عدد):
    ارجع عدد ** 3
```

```arabic
# رئيسي.apy
استورد helper كـ مساعد

اطبع(مساعد.مربع(5))   # 25
اطبع(مساعد.مكعب(3))   # 27
```

شغّل `apython رئيسي.apy` وسيجد الخطّاف ملف `helper.apy` ويترجمه ويجعله قابلاً للاستيراد.

الكلمتان المفتاحيتان هنا هما `استورد` (`import`) و`كـ` (`as`). يمكنك أيضاً استيراد أسماء بعينها:

```arabic
من helper استورد مربع
اطبع(مربع(6))   # 36
```

وحدات المكتبة القياسية (`math` و`json` و`os` وغيرها) تُستورَد بأسمائها الإنجليزية المعتادة:

```arabic
استورد math
اطبع(math.pi)
```

المرحلة الأولى (أ) تترجم *اللغة* فقط وليس المكتبة القياسية. ترجمة أسماء المكتبات هدف المرحلة الثانية (ب)؛ انظر
[`decisions/0008-phase-b-charter.md`](../../decisions/0008-phase-b-charter.md).

**ملاحظة معروفة:** `from . import x` داخل ملف `__init__.apy` لا يترجم حالياً. الحل البديل: `استورد pkg.sub كـ sub`.

انظر: [`examples/07_imports.apy`](../../examples/07_imports.apy)

## ٩. قراءة رسائل الخطأ

الاستثناءات غير المُعالَجة تطبع تتبعاً للأخطاء بالعربية:

```bash
$ apython -c '1 / 0'
تتبع_الأخطاء (المكدس الأحدث آخرا):
  ملف "<string>", سطر 1, في <الوحدة>
خطأ_القسمة_على_صفر: القسمة على صفر
```

البنية مطابقة لتتبع الأخطاء في بايثون، مع تسميات مترجمة:

| بايثون | apython |
|---|---|
| `Traceback (most recent call last):` | `تتبع_الأخطاء (المكدس الأحدث آخرا):` |
| `File "..."` | `ملف "..."` |
| `line N` | `سطر N` |
| `in <module>` | `في <الوحدة>` |

أسماء الاستثناءات مترجمة بصورة منهجية. `ZeroDivisionError` تصبح `خطأ_القسمة_على_صفر`؛ و`KeyError` تصبح `خطأ_مفتاح`؛ و`ValueError` تصبح `خطأ_قيمة`. البادئة `خطأ_` تقابل اللاحقة `Error` في بايثون. بعض الاستثناءات التي لا تنتهي بـ `Error` في بايثون لا تُضاف إليها هذه البادئة عربياً: `KeyboardInterrupt` و`StopIteration` و`Warning` و`GeneratorExit` و`SystemExit`.

القائمة الكاملة لـ 38 نوع استثناء مترجماً في
[`dictionaries/exceptions-ar-v1.md`](../../dictionaries/exceptions-ar-v1.md).

عند رفع استثناء خاص بك يمكنك استخدام الاسم العربي أو الإنجليزي — كلاهما يُحلّ إلى الكائن ذاته:

```arabic
حاول:
    ارفع خطأ_قيمة("القيمة غير صالحة")
استثناء خطأ_قيمة كـ خطأ:
    اطبع(f"التُقط: {خطأ}")
```

المخرجات:

```
التُقط: القيمة غير صالحة
```

نحو ثلاثين من أكثر رسائل المترجم شيوعاً مترجمة أيضاً. الرسائل التي لا يوجد لها ترجمة تمر كما هي بالإنجليزية — خطأ إنجليزي صريح أفضل من رسالة عربية مضللة.

## ١٠. إلى أين بعد ذلك؟

- **شغّل الأمثلة.** [`examples/01_hello.apy`](../../examples/01_hello.apy) حتى `07_imports.apy` برامج تدريجية تغطي كل ميزات اللهجة. كل مثال قصير متعمداً؛ معاً تمارس كل شيء.

- **تصفّح القاموس.** [`dictionaries/ar-v1.md`](../../dictionaries/ar-v1.md) هو القائمة المرجعية الكاملة. لا داعي لحفظه، لكن التصفح السريع يبني خريطة ذهنية مفيدة.

- **جرّب الوضع التفاعلي.** شغّل `apython` بلا وسائط. يقبل نفس الصياغة العربية تماماً، مع إمكانية الإدخال متعدد الأسطر.

- **ابلغ عما أدهشك.** القاموس سيتطور في إصدار `ar-v2`. إذا قرأت كلمة مفتاحية وبدت غريبة أو غير طبيعية، فهذه معلومة تستحق التوثيق.

apython لهجة تعليمية أولاً وأداة إنتاجية ثانياً (راجع [`decisions/0007-scope.md`](../../decisions/0007-scope.md) لتفاصيل هذا الاختيار المقصود). مستقبل المرحلة الثانية — ومنها مسألة ما إذا كان `استورد طلبات` سيعني `import requests` يوماً — مُخطَّط له في [`decisions/0008-phase-b-charter.md`](../../decisions/0008-phase-b-charter.md).
