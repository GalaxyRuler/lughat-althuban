# نظرة عامة على مشروع لغة الثعبان

**لغة الثعبان** لهجة برمجية عربية مبنية فوق بايثون. تكتب الكلمات المفتاحية والدوال المدمجة والاستثناءات بالعربية، وتُترجم الملفات إلى بايثون القياسي في وقت التحميل وتُنفَّذ بواسطة CPython — دون تشعيب للمترجم أو تعديل على البنية التركيبية للغة.

---

## ما هو لغة الثعبان؟

لغة الثعبان هو **معالج مسبق قائم على الرموز المعجمية**. عندما تشغّل ملف `.apy` يمر بالمراحل التالية:

```
المدخل العربي (.apy)
        ↓
pretokenize  ← تحويل الأرقام العربية، استبدال علامات الترقيم، رفض محارف bidi المحظورة
        ↓
tokenize     ← مُرمِّز بايثون القياسي
        ↓
translate    ← إعادة كتابة أسماء المعرّفات العربية إلى نظيراتها الإنجليزية
        ↓
untokenize + compile + exec
        ↓
تنفيذ CPython عادي
```

لا يوجد تعديل على شجرة AST ولا تشعيب لـ CPython. المشروع مُصمَّم معمارياً على غرار مشروع [zhpy](https://github.com/zhpy/zhpy) للصينية.

---

## مثال سريع

```python
# مرحبا.apy
دالة مرحبا(اسم):
    اطبع(f"مرحبا يا {اسم}")

لكل شخص في ["سارة", "أحمد", "ليلى"]:
    مرحبا(شخص)
```

```bash
$ ثعبان مرحبا.apy
مرحبا يا سارة
مرحبا يا أحمد
مرحبا يا ليلى
```

يتطلب بايثون 3.11 أو أحدث.


## طريقة التشغيل

أربع واجهات دخول تعكس واجهة `python` القياسية:

| الأمر | الوصف |
|-------|--------|
| `ثعبان ملف.apy` | تشغيل ملف |
| `ثعبان -c 'اطبع("مرحبا")'` | تنفيذ سلسلة نصية مباشرة |
| `ثعبان - < ملف.apy` | القراءة من المدخل القياسي |
| `ثعبان` | فتح بيئة تفاعلية REPL |
| `ثعبان ترجمة-عكسية ملف.py` | تحويل ملف Python إلى `.apy` اعتماداً على المعجم العكسي |

```bash
ثعبان --help     # عرض التعليمات
ثعبان --version  # عرض رقم الإصدار
```

رموز الخروج: `0` نجاح، `1` خطأ تشغيل أو ترجمة، `2` خطأ استخدام.

## استيراد ملفات `.apy`

يُثبِّت لغة الثعبان خطّاف استيراد في `sys.meta_path` يتيح استيراد ملفات `.apy` كأي وحدة بايثون عادية:

```python
# main.apy
استورد helper باسم مساعد
اطبع(مساعد.مربع(5))
```

```python
# helper.apy
دالة مربع(س):
    ارجع س ** 2
```

```bash
$ ثعبان main.apy
25
```

تعمل الحزم المختلطة (`.py` + `.apy`) بشكل طبيعي. يرى ملف `.py` الذي يستورد وحدة `.apy` الأسماء بصيغتها المُطبَّعة (المُنوَّرمة) وفق ADR 0004.

## رسائل الخطأ بالعربية

تصدر رسائل الخطأ غير المُعالَجة بالعربية تلقائياً:

```bash
$ ثعبان -c '1 / 0'
تتبع_الأخطاء (المكدس الأحدث آخراً):
  ملف "<string>", سطر 1, في <الوحدة>
خطأ_قسمة_صفر: القسمة على صفر
```

يترجم لغة الثعبان 38 نوعاً من الاستثناءات القياسية ونحو 30 رسالة شائعة من رسائل المترجم. الأنواع والرسائل غير المُعرَّفة تمر دون تغيير.

في المرحلة د أصبحت أسماء الاستثناءات تأتي من `lexicon/core.toml`، وقوالب الرسائل من `lexicon/messages.toml`. يمكن اختيار نمط الأثر من CLI أو من المتغير `PYTHONTRACEBACK`:

```bash
ثعبان --tracebacks=arabic ملف.apy   # الافتراضي
ثعبان --tracebacks=english ملف.apy  # أثر Python الأصلي
ثعبان --tracebacks=mixed ملف.apy    # اسم عربي ورسالة إنجليزية
```

## قاموس الكلمات المفتاحية

القاموس الكامل للكلمات المفتاحية والدوال المدمجة والاستثناءات موجود في `lexicon/core.toml`، وتُولَّد منه نسخ القواميس في:

| الملف | الوصف |
|-------|--------|
| `lexicon/core.toml` | مصدر الحقيقة للكلمات المفتاحية والدوال والاستثناءات |
| `dictionaries/ar-v2.md` و`arabicpython/dictionaries/ar-v2.md` | القاموس الافتراضي الحالي |
| `dictionaries/ar-v1.1.md` | قاموس التوافق السابق |
| `dictionaries/ar-v1.md` | القاموس التاريخي المجمد |
| `lexicon/messages.toml` | مصدر رسائل الأدوات وقوالب رسائل الأثر |

الإصدار `ar-v1` مجمَّد وفق ADR 0008 §B.0. أي تغيير على الكلمات القائمة يبدأ من `lexicon/` ثم يُولَّد إلى القواميس والوثائق.

**عينة من الكلمات المفتاحية:**

| بايثون | عربي | بايثون | عربي |
|--------|-------|--------|-------|
| `if` | `إذا` | `for` | `لكل` |
| `def` | `دالة` | `class` | `صنف` |
| `return` | `ارجع` | `import` | `استورد` |
| `print` | `اطبع` | `True` | `صحيح` |
| `False` | `خطأ` | `None` | `لاشيء` |

للاطلاع على القائمة الكاملة راجع [المعجم العربي الموحد](lexicon.md).

## المعمارية في فقرة واحدة

يمر المدخل عبر `pretokenize` (طيّ الأرقام العربية-الهندية إلى ASCII، استبدال علامات الترقيم، رفض 12 محرفاً من محارف bidi وفق UAX #9) ← `tokenize` الخاصة ببايثون ← مُعيد كتابة الأسماء الذي يستشير قاموس عربي→بايثون المجمَّد ← `untokenize` ← `compile` ← `exec`. لا تعديل على شجرة AST، ولا تشعيب لـ CPython. المسار ذاته يدعم جميع واجهات الدخول (CLI، REPL، خطّاف الاستيراد).

## الحدود المعروفة (المرحلة A)

- **`from . import x` داخل `__init__.apy`** لا تُترجم — الحلّ: `import pkg.sub as sub`.
- **الوصول إلى الخصائص من `.py` إلى `.apy`** يتطلب الشكل المُطبَّع (المُنوَّرم) وفق ADR 0004.

## الشكر والاعتراف

- **zhpy** (gasolin، 2014) — لهجة بايثون الصينية التي يعتمد عليها هذا المشروع معمارياً.
- **قلب (Qalb)** لرمزي ناصر — النقد الذي يأخذه هذا المشروع بجدية، وسبب جعل المرحلة B مرحلة مستقلة مؤجلة بدلاً من كونها الهدف الافتراضي.
- **Hedy** — دليل على أن بايثون التعليمية متعددة اللغات والتدريجية تعمل على نطاق واسع.
