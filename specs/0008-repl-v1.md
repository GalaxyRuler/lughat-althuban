# Spec Packet 0008: repl-v1

**Phase**: A
**Depends on**: Packet 0005 (translate); Packet 0006 (cli); Packet 0007 (import-hook); ADR 0001 (architecture), ADR 0007 (scope).
**Estimated size**: medium (one focused implementer session)
**Owner**: Gemini 3.1 Flash

## Goal

Implement an interactive Arabic Python REPL: a learner runs `apython` with no arguments and gets a Python-style prompt where they can type Arabic statements, see them executed, and build up state across lines. Multi-line constructs (functions, classes, `if/for/while` blocks) work via a continuation prompt. Variables, imports, and definitions persist across the session.

This completes the "I can write Arabic Python at a prompt" learner experience that mirrors stock `python`. After this packet lands, `apython` matches the four canonical entry points of stock Python: `apython file.apy`, `apython -c CODE`, `apython -` (stdin), and `apython` (REPL).

## Non-goals

- **No async/await REPL support.** Python 3.8+ supports `async` at the top level via `python -m asyncio`; we do NOT replicate that. A learner who needs `async` can run code from a file.
- **No multiline editing.** Each line is a separate `input()` call. No arrow-key navigation across already-submitted lines (stock `python` REPL has the same limitation; `bpython`/`ptpython` add it). Out of scope.
- **No syntax highlighting at the prompt.** Plain text input. (Out of scope; would require a third-party dep.)
- **No paste-mode detection.** If a user pastes a 50-line block, it is processed line-by-line just like stock `python`. They can wrap it in `exec("""...""")` if they need atomic execution.
- **No `%magic` commands** (IPython-style). Out of scope.
- **No persistent variable inspection** beyond what stock `python` offers (`dir()`, `locals()`).
- **No translated tracebacks.** Exceptions still print English message text. Translated tracebacks are Packet 0009.
- **No Arabic prompt strings by default.** `sys.ps1 = ">>> "` and `sys.ps2 = "... "` to match stock Python. Users can override via env var (see "Environment overrides"). The choice to keep ASCII prompts is deliberate: the prompts are recognized worldwide as "Python prompt"; reinventing them adds confusion for learners who later move to stock `python`.
- **No bytecode caching, no `-i` flag (run file then drop to REPL), no startup-script support (`PYTHONSTARTUP`).** Out of scope; revisit if learners ask.

## Public interface

```python
# arabicpython/repl.py

import code as _code_module


class ArabicConsole(_code_module.InteractiveConsole):
    """An InteractiveConsole that translates Arabic source before compiling."""

    def __init__(
        self,
        locals: "dict | None" = None,
        filename: str = "<stdin>",
    ) -> None:
        ...

    def runsource(
        self,
        source: str,
        filename: str = "<stdin>",
        symbol: str = "single",
    ) -> bool:
        """Translate, then compile-or-defer-or-error.

        Returns:
            True  → input is incomplete; caller should request another line.
            False → input was processed (executed OR errored). Caller resets the buffer.
        """


def run_repl(
    *,
    banner: "str | None" = None,
    exit_msg: "str | None" = None,
) -> int:
    """Start an interactive Arabic Python session.

    Sets up readline (if available) for history and tab completion, prints the
    banner, runs the InteractiveConsole loop until EOF, returns 0.

    Args:
        banner: override the default startup banner (mostly for tests).
        exit_msg: override the default exit message (mostly for tests).

    Returns:
        0 on clean exit (EOF or sys.exit(0)).
        Other ints if the user calls sys.exit(N).
    """
```

## Behavior

### `ArabicConsole.runsource`

The standard `code.InteractiveConsole.runsource` does roughly:

```python
def runsource(self, source, filename="<input>", symbol="single"):
    try:
        code = self.compile(source, filename, symbol)
    except (SyntaxError, OverflowError, ValueError):
        self.showsyntaxerror(filename)
        return False
    if code is None:
        return True   # incomplete → ask for more
    self.runcode(code)
    return False
```

Our override inserts a translate step:

```python
def runsource(self, source, filename="<stdin>", symbol="single"):
    try:
        translated = translate(source)
    except SyntaxError as e:
        # Distinguish "incomplete input" from "real translate-time error".
        # tokenize.TokenError surfaces as SyntaxError with msg like
        # "EOF in multi-line statement" / "EOF in multi-line string" — that
        # means the user is mid-statement and we want a continuation prompt.
        if _is_incomplete_marker(e):
            return True
        self._write_translate_error(e, filename)
        return False

    return super().runsource(translated, filename, symbol)
```

Where `_is_incomplete_marker(e)` is a small helper that checks `str(e)` for substrings like `"EOF in multi-line"` and the standard `compile_command`-style markers. Implementer: study what stock CPython's `code.compile_command` looks for (it inspects `SyntaxError.msg` for specific markers); mirror that logic for *translate-time* errors. Bidi-rejection and mixed-digit errors are NOT incomplete-markers — those are real errors and should reset the buffer.

`_write_translate_error` prints a one-line, traceback-free error to `self.write` (which routes to stderr in the REPL by default):

```
  File "<stdin>", line 1
    {e.msg}
```

Do NOT print a Python traceback for translate-time errors at the REPL — they are user input mistakes, not crashes.

### `run_repl`

Pseudocode:

```python
def run_repl(*, banner=None, exit_msg=None) -> int:
    _setup_readline()  # best-effort; never raises

    if banner is None:
        banner = (
            f"apython {__version__} — Arabic Python REPL\n"
            f"Python {sys.version.split()[0]} on {sys.platform}\n"
            f'Type "help()", "exit()", or press Ctrl-D to quit.\n'
        )
    if exit_msg is None:
        exit_msg = ""  # match stock python (silent exit)

    sys.ps1 = getattr(sys, "ps1", ">>> ")
    sys.ps2 = getattr(sys, "ps2", "... ")

    console = ArabicConsole()
    try:
        console.interact(banner=banner, exitmsg=exit_msg)
    except SystemExit as e:
        if e.code is None:
            return 0
        if isinstance(e.code, int):
            return e.code
        sys.stderr.write(str(e.code) + "\n")
        return 1
    return 0
```

`InteractiveConsole.interact` already:
- Prints the banner.
- Loops `input(sys.ps1)` / `input(sys.ps2)`.
- Catches `KeyboardInterrupt` → resets the buffer and prints `KeyboardInterrupt` (without exiting).
- Catches `EOFError` → exits the loop cleanly.

We do not need to reimplement these.

### `_setup_readline`

```python
def _setup_readline() -> None:
    """Best-effort readline setup. Never raises."""
    try:
        import readline
        import rlcompleter  # noqa: F401 — registers default Completer
    except ImportError:
        return  # Windows without pyreadline3, embedded environments, etc.

    # History
    history_path = os.path.expanduser("~/.apython_history")
    try:
        readline.read_history_file(history_path)
    except (OSError, FileNotFoundError):
        pass  # first run or unreadable; ignore

    # Truncate to 1000 lines on exit. atexit ensures we save even on EOF.
    readline.set_history_length(1000)
    atexit.register(_save_history, history_path)

    # Tab completion. The default rlcompleter completes from the current
    # interactive namespace, which is what we want.
    readline.parse_and_bind("tab: complete")


def _save_history(path: str) -> None:
    try:
        import readline
        readline.write_history_file(path)
    except Exception:
        pass  # disk full, permission, whatever — don't crash on shutdown
```

Tab completion in v1 uses the default `rlcompleter` against the current `console.locals`. This means:
- Arabic identifiers the user has *defined* (`اسم = ٥`) complete via TAB.
- Arabic *keywords* (`إذا`, `دالة`, …) do NOT complete in v1 — they're not in any namespace and `rlcompleter` is prefix-based, which doesn't play well with Arabic word completion.
- Stock Python builtins (`print`, `len`, …) complete normally.

Adding Arabic-keyword completion is a v1.1 nicety (would need a custom completer that pulls from `dialect.names`).

### Environment overrides

Two env vars, both optional:

- `APYTHON_PS1` — overrides `sys.ps1`. If set, used as the primary prompt.
- `APYTHON_PS2` — overrides `sys.ps2`. If set, used as the continuation prompt.

If unset, defaults to `">>> "` / `"... "`. Set these in `run_repl` before calling `console.interact`.

History file path is fixed at `~/.apython_history` in v1. (We can add an env var override later if anyone asks.)

### CLI integration

In `arabicpython/cli.py`, replace the current "no args → print usage, exit 2" branch with:

```python
else:
    # No FILE, no -c, no '-': drop into the REPL.
    from arabicpython.repl import run_repl
    return run_repl()
```

The REPL branch fires regardless of TTY-ness. If a user pipes input into `apython` with no args (e.g., `echo "اطبع(1)" | apython`), they get an interactive console reading that piped input, which exits at EOF. This matches stock `python`'s behavior.

The `--version` and `-h` flags continue to short-circuit before this branch, exactly as today.

The deprecated test `test_no_args_prints_usage_exits_two` in `tests/test_cli.py` MUST be replaced (not deleted; replace with the new behavior) by:

```python
def test_no_args_drops_into_repl(monkeypatch, capsys):
    """main([]) drops into REPL; we feed an immediate EOF to exit cleanly."""
    monkeypatch.setattr("sys.stdin", io.StringIO(""))
    ret = main([])
    assert ret == 0
    out, err = capsys.readouterr()
    assert "apython" in out or "apython" in err
```

## Implementation constraints

- **Dependencies**: stdlib only (`code`, `readline` (best-effort), `rlcompleter`, `atexit`, `os`, `sys`).
- **Python version**: 3.11+.
- **No global state mutation outside the REPL session**: `sys.ps1`/`sys.ps2` are session-scoped (Python's REPL sets them too — we do the same).
- **Test isolation**: REPL tests must not actually open a `readline` history file in the user's home directory. Monkeypatch `os.path.expanduser` or set the history path via test fixture.
- **Style**: pass `ruff check .` and `black --check .` at project defaults (line length 100).
