## Implementation notes worth remembering — non-obvious decisions

- **Collision Mitigation in Tests**: In `test_run_class_with_method`, the common Arabic letter `ك` was initially used as a variable name but failed because it maps to the Python keyword `as` (via `كـ` normalization). I changed the variable name to `كائن` (object) to avoid this syntax error.
- **Path Escaping in Tests**: On Windows, `repr(list)` escapes backslashes in paths. I updated `test_sys_argv_in_file_mode` to construct the expected string using `str([path, ...])` which correctly accounts for this escaping when compared against `capsys` output.
- **BOM Handling**: I chose to rely on `translate` (and its internal `tokenize` call) to handle the UTF-8 BOM. Added a test case `test_utf8_with_bom_handled` to verify this works correctly.
